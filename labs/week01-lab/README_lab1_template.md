# Lab 1: เริ่มต้นใช้งาน GitHub และ Python

## จุดประสงค์ของแลป

แลปนี้ออกแบบมาเพื่อให้นิสิตที่ยังไม่เคยใช้ Git มาก่อนสามารถสร้าง repository จาก template ของรายวิชา ดาวน์โหลดโปรเจกต์ลงเครื่องในแลป แก้ไขไฟล์ Python และส่งงานขึ้น GitHub ได้สำเร็จภายในคาบแรกเมื่อจบแลปนี้ นิสิตควรเข้าใจคำพื้นฐานอย่าง repository, commit และ push ในระดับใช้งานจริงสำหรับรายวิชา

## สิ่งที่ต้องมี

ก่อนเริ่ม ให้เตรียมบัญชี GitHub ที่ยืนยันอีเมลแล้ว และตรวจสอบว่าเครื่องในแลปมี Git, Python และ Visual Studio Code พร้อมใช้งานรายวิชานี้จะใช้ Python เป็นภาษาหลักสำหรับหัวข้อโครงสร้างข้อมูล เช่น lists, stacks, queues, linked lists, trees และ graphs

## โครงสร้างไฟล์ใน template

ใน repository template นี้จะมีไฟล์อย่างน้อย 3 ไฟล์ คือ `README.md`, `hello.py` และ `simple_calc.py` เพื่อใช้ฝึกทั้งการเปิดโปรเจกต์ การรัน Python และการแก้ไขโค้ดเบื้องต้น Python lists และโครงสร้างข้อมูลพื้นฐานอื่น ๆ จะถูกใช้ต่อเนื่องในรายวิชาหลังจากแลปนี้

## ขั้นตอนที่ 1: สร้าง repository ของตนเองจาก template

1. เปิดลิงก์ GitHub template ของรายวิชา
2. กดปุ่ม **Use this template** หรือ **Generate from template**
3. ตั้งชื่อ repository ตามรูปแบบที่ผู้สอนกำหนด เช่น `dst-lab01-รหัสนิสิต`
4. กดสร้าง repository

หลังจากสร้างเสร็จ นิสิตจะได้ repository ของตนเองที่มีไฟล์เริ่มต้นเหมือนกันทุกคน

## ขั้นตอนที่ 2: ดาวน์โหลดโปรเจกต์ลงเครื่องในแลป

คัดลอก URL ของ repository ของตนเองแบบ HTTPS แล้วเปิด Terminal หรือ Git Bash จากนั้นใช้คำสั่งต่อไปนี้

```bash
git clone <URL-repository-ของตนเอง>
cd <ชื่อ-repository>
```

คำสั่ง `git clone` ใช้สำหรับคัดลอก repository จาก GitHub มายังเครื่องในแลป และทำให้สามารถแก้ไขไฟล์ในเครื่องได้

## ขั้นตอนที่ 3: เปิดโปรเจกต์และรัน Python

เปิดโฟลเดอร์โปรเจกต์ด้วย Visual Studio Code แล้วเปิดไฟล์ `hello.py` จากนั้นรันคำสั่งต่อไปนี้ใน Terminal

```bash
python hello.py
```

ถ้าระบบแสดงข้อความจากโปรแกรมได้ แปลว่าเครื่องพร้อมสำหรับการเขียน Python แล้ว Python เหมาะกับการเริ่มเรียนโครงสร้างข้อมูลเพราะไวยากรณ์อ่านง่ายและมีชนิดข้อมูลแบบ collection ให้ใช้งาน เช่น list, tuple และ dictionary

## ขั้นตอนที่ 4: แก้ไขไฟล์ `hello.py`

ให้แก้ไฟล์ `hello.py` เพื่อแสดงข้อมูลของตนเองอย่างน้อย 3 บรรทัด ได้แก่ ชื่อ-นามสกุล รหัสนิสิต และข้อความต้อนรับเข้าสู่รายวิชา ตัวอย่างรูปแบบผลลัพธ์เช่น

```text
Name: Somchai Jaidee
Student ID: 66123456
Welcome to Data Structures with Python
```

กิจกรรมนี้ช่วยให้คุ้นกับคำสั่ง `print()` และการบันทึกไฟล์ Python เบื้องต้น

## ขั้นตอนที่ 5: ทดลองไฟล์ `simple_calc.py`

เปิดไฟล์ `simple_calc.py` แล้วรันโปรแกรมเพื่อลองรับค่าตัวเลข 2 ค่าและแสดงผลรวมการใช้ตัวแปร รับข้อมูลเข้า และคำนวณพื้นฐาน เป็นจุดเริ่มต้นสำคัญก่อนเรียนโครงสร้างข้อมูลที่ซับซ้อนขึ้น

```bash
python simple_calc.py
```

จากนั้นให้แก้โปรแกรมเพิ่มเติม เช่น แสดงผลต่างหรือผลคูณเพิ่มอีก 1 ค่า เพื่อฝึกการแก้ไขโค้ดด้วยตนเอง

## ขั้นตอนที่ 6: บันทึกงานด้วย Git

หลังจากแก้ไขไฟล์แล้ว ให้ใช้คำสั่งต่อไปนี้เพื่อดูสถานะไฟล์ เพิ่มไฟล์เข้าสู่การเตรียม commit และบันทึกการเปลี่ยนแปลง

```bash
git status
git add .
git commit -m "lab01: finish first python edits"
```

`git status` ใช้ดูว่าไฟล์ใดถูกแก้ไขอยู่ ส่วน `git commit` คือการบันทึก snapshot ของงานในเวลาหนึ่ง

## ขั้นตอนที่ 7: ส่งงานขึ้น GitHub

เมื่อ commit เรียบร้อยแล้ว ให้ส่งงานขึ้น GitHub ด้วยคำสั่งต่อไปนี้

```bash
git push origin main
```

หลังจากนั้นให้เปิดหน้า repository บน GitHub และตรวจสอบว่าไฟล์เวอร์ชันล่าสุดของตนเองถูกอัปโหลดขึ้นไปแล้ว

## งานที่ต้องส่ง

ให้นิสิตส่ง URL ของ repository ของตนเองผ่านช่องทางที่ผู้สอนกำหนด เช่น MS Teams หรือแบบฟอร์มของรายวิชา พร้อมตรวจสอบว่าใน repository มีไฟล์ที่แก้ไขแล้วและมี commit อย่างน้อย 1 ครั้ง

## ปัญหาที่พบบ่อย

ถ้ารัน `python hello.py` ไม่ได้ ให้ตรวจสอบว่า Python ถูกติดตั้งและเรียกใช้งานได้จาก Terminalถ้า `git push` ไม่สำเร็จ ให้ตรวจสอบว่า login GitHub ถูกต้อง และกำลังส่งไปยัง repository ของตนเอง ไม่ใช่ของเพื่อน

## คำสั่งที่ใช้ในแลปนี้

```bash
git clone <URL>
git status
git add .
git commit -m "message"
git push origin main
python hello.py
python simple_calc.py
```

คำสั่งชุดนี้เพียงพอสำหรับแลปแรก และยังไม่จำเป็นต้องใช้ branch, merge หรือคำสั่งขั้นสูงอื่น ๆ
